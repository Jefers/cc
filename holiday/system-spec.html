Overview

The Philippine Adventure Planner is a static web application hosted on GitHub Pages, designed to manage travel plans for a trip to the Philippines. It uses HTML, CSS, and JavaScript with local storage for data persistence.

File Structure

  • index.html: Main hub with navigation and data export/import.
  • hotel-tracker.html: Hotel booking management.
  • realtime-tracker.html: Flight detail tracking.
  • budget-tracker.html: Cost tracking.
  • booking-guide.html: Booking instructions.
  • user-guide.html: User instructions.
  • system-spec.html: This specification.
  • style.css: Centralized styling.
  • script.js: Shared export/import functions.
  • assets/: Folder for static images.

Features

Hotel Tracker

Purpose: Manage hotel bookings.

Data Fields: Hotel name, booking ID, arrival time, Google Plus Code, notes, check-in/check-out times.

Functions: Add, edit, delete, add to Google Calendar.

Storage: localStorage keys: manilaHotel, boholHotel, etc.

Real-Time Tracker

Purpose: Track flight details.

Data Fields: Flight number, departure time, arrival time.

Functions: Add, edit, delete, add to Google Calendar.

Storage: localStorage keys: dublinManila, manilaBohol, etc.

Budget Tracker

Purpose: Monitor trip expenses.

Data Fields: Costs for bookings and extras (e.g., food, taxis).

Functions: Input costs, calculate running total.

Storage: localStorage keys: dublinManila, activities, etc.

Data Sharing

Purpose: Share data between devices.

Functions: Export all localStorage to travel-data.json, import from JSON file.

Location: index.html.

Technical Details

  • Storage: Browser localStorage for client-side persistence.
  • Styling: style.css with a pastel color scheme (e.g., #ff6f61 coral, #f0f4f8 light blue).
  • Scripts:
    • script.js: Export/import functions.
    • Page-specific JS for adding/editing/deleting entries.
  • External Integration: Google Calendar via URL generation, Google Drive for manual image links.

Limitations

  • No real-time sync; data sharing requires manual export/import.
  • Static hosting (GitHub Pages) limits dynamic features.
  • Images require manual HTML edits for Google Drive links.
